# Technology Stack

本文件详细说明了 "auto-scanner" 项目所采用的技术栈及其选择理由。

## 1. 核心语言与运行时
*   **编程语言**: Rust
    *   **理由**: 提供卓越的性能、内存安全性和强大的二进制分发能力，非常适合开发高可靠性的自动化工具。
*   **异步运行时**: `tokio`
    *   **理由**: Rust 生态中最成熟、性能最强的异步运行时，拥有最广泛的库支持。

## 2. 自动化与爬虫技术
*   **浏览器自动化**: `playwright-rust` (或类似的 Rust Playwright 绑定)
    *   **理由**: Playwright 提供强大的 API 来模拟复杂的人工登录操作，并能有效处理现代 Web 框架。

## 3. 数据存储
*   **数据库**: SQLite
    *   **理由**: 无需独立服务器，易于分发，完美契合个人工具的本地存储需求。
*   **数据库驱动/工具**: `sqlx` (具有 `sqlite` 特性)
    *   **理由**: 提供全异步支持和编译时 SQL 检查，确保数据操作的类型安全和高性能。

## 4. 命令行交互 (CLI)
*   **参数解析**: `clap` (带有 `derive` 特性)
    *   **理由**: Rust 社区的标准 CLI 工具，能够快速构建直观且功能丰富的命令行界面。

## 5. 其他关键库 (预估)
*   **日志系统**: `tracing` 及其生态（如 `tracing-subscriber`）
*   **CSV 解析**: `csv` crate
*   **配置管理**: `serde` (用于数据序列化和反序列化)
*   **错误处理**: `anyhow` 或 `thiserror`
